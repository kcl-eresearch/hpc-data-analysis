PART 5: Requested vs Allocated Memory (tres_req vs tres_alloc)
================================================================================

Data sources (all from create_job_table):
  - tres_req (TRES ID 2): total requested memory in MB
  - tres_alloc (TRES ID 2): total allocated memory in MB
  - mem_req (bit 63): distinguishes --mem vs --mem-per-cpu
  - cpus_req vs alloc_cpus (TRES ID 1 in tres_alloc): requested vs allocated CPUs

Hypothesis: For --mem-per-cpu jobs where alloc_cpus > cpus_req,
Slurm allocates proportionally more memory than requested.
This causes memory efficiency > 100% even with ConstrainRAMSpace enabled,
because the cgroup enforces the *allocated* amount, not the *requested* amount.
